Creative programing

Using Processing


My first interaction with programing was in a course "Programing bootcamp" where we started learning creative programing with the platform "Processing". We had 5 days of the bootcamp in which we were learning a new skill every day and doing a small project after the lecture. In the evening we would present those projects and get feedback on our progress.

Day one:

The first day was relatively easy considering the fact that it was my first-time coding. We have learned how to draw a line, rectangle, triangle, an ellipse and how to add a randomizing function, fill and stroke. After learning it was nice to at least have the basic understanding of how creative code is written. When we got to the assignment which was to create a static image with at least one element which is random but at the same time similar – I wanted to create something recognizable and a little bit more difficult than a shape with randomly changing color. So, I decided to make a Christmas tree with randomly placed snowflakes around it. I also wanted to make the snowflakes size different. In addition, I made the color of the ground random shade of grey between a specific parameter.

Day two:

I have learned how to create a moving image in other words how to create the loop that program is always running with the help of void draw command. Also, I have learned how to utilize if function which allowed me to create something more engaging.So, when the timecame to create my own game, I felt extremely incompetent. The game I created was easy and simple, a yellow dot which is the avatar of the game has to eat two green ellipses (broccolis) to vin the game. Also, after consumption the avatar increases size and after eating both broccolis a player is greeted with a victory screen.
After learning how to do this I felt wery inspired and wanted to create more.

Day three:

On this day we have learned while and for loops. That helped us to "run" an action continuously, until a specific parameter or condition was met. This allowed us to create multiple repeating shapes or lines by only writing few lines of code instead of coding every individual object. At this point programing became fun again because I understood things and how to make them happen on the screen. When it was time to make an interactive artwork I decided to make an illusion of a tunnel by drawing multiple increasing circles. The interactivnes of the art-work is changing center of the tube illusion. The mouse cursors placement on the screen was the center of the tube.

Day four:

On a day four we have learned how to use arrays. Learning arrays was quite fun and nice. They are useful when dealing with a lot of variables under the same name and allows the programmer to make a shorter and more manageable code.When it came time to do the project of the day which was a game, my idea was to create a frustrating game. I wanted to make hard to control avatar which moves across the screen faster than the user can notice. The goal of the game is to reach right side of the screen and avoid hidden obstacles that show themselves within close proximity of the avatar. When stepped on an obstacle it turns red and starts collecting anti points. The whole point of the game is to collect as les of them as possible. It was really fun working on a thing like that and I haven’t even noticed how the time passed. At this point I was surprised that I actually enjoy programing.

Day five:

On a day five we had to combine all the skills that we have learned and create something awesome. Some people made games, others made artworks and I have decided to work on an artwork myself. I was a little bit disappointed of my previous one, because it was too simple, and I have struggled to come up with an idea. So through the duration of the day I created a moving artwork that can be refreshed and set of in a different direction and can e colored with the help of a c button. I was happy with my work because the artwork resembled a logo.
